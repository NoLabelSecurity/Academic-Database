We will be using XAMPP with the MySQL module to create and populate a database from within the terminal on Windows OS. There will be a total of 5 MySQL scripts that will populate the database by creating tables, inserting static data, inserting dummy (test) data which will come from a previous assignment, insert my personal information (my student data), and generate a report once the database is completed. 

We will start by opening the XAMPP client and starting the MySQL module then opening a terminal and entering into MySQL Marina DB. After creating a database which we will call five_scripts we will select to use the database. From that point we will use the source command with the proper file directory address to execute the schema_definition script to create the tables. 

The schema_definition.sql creation script should create the tables in the correct order so that the Query will OK for each table. But from the terminal the show tables command should display all 25 tables in alphabetical order starting with Address and Ending with ZipCode, regardless of how they are ordered in the script. 

The next script, seed_static_data.sql, will populate static (unchanging) data into the database accurately and consecutive to the creation scripts execution. The static data values will be data relating to the tables and fields. (create relevant static data for each table and every field within them.)

The third script, dummy_data_pop.sql, will populate the database with dummy (test) data with the four students listed in the previous normalization exercise. (reference "dummy data")
